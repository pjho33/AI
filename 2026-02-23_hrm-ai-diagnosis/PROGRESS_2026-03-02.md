### 1. LES 채널 선택 알고리즘 v3 확정 ✅

**배경 및 문제점**:
- 기존 알고리즘(v2)은 체부 수축파 원위부 끝(distal end)을 단순 peak 기반으로 판단
- 문제: 횡격막 각(diaphragmatic crus) 채널이 호흡성 진동(oscillation)으로 인해 높은 압력 기록
- 결과: LES 탐색 범위가 너무 낮게 설정되어 실제 LES 고압대를 놓침

**해결 방안 (v3 알고리즘)**:

#### 단계 1: 참조 삼킴(swallow) 선택
- 전체 삼킴 중 체부(ch5~27) 수축파 강도 상위 3개 선택
- 근거: 수축파가 가장 깊이 내려간 삼킴 = 체부 원위부 끝이 가장 정확히 반영됨
- 이중 삼킴(double swallow)은 자동 제외

#### 단계 2: 수축파 원위부 끝 탐지
- 탐색 범위: ch25~34 (골격근/평활근 전환 이후 평활근 구간)
- 첫 도달(first_hit) 방식: 각 채널에서 기저선+10mmHg 첫 돌파 시점 계산
- 단조 진행 원칙: 첫 도달 시간이 순차적으로 증가하는 마지막 채널
- 허용 오차: 0.5초 (연속 2회 역전/없음 발생 시 종료)
- 외부 압박 채널 필터: 삼킴 시작 후 1초 이내 발화 → 제외

#### 단계 3: LES 탐색 범위 설정
```python
les_lo = max(distal_end_ch + 1, 28)  # 최소 ch28
les_hi = min(les_lo + 4, 34)         # ch33까지만 (ch34는 위 경계)
```

#### 단계 4: LES 후보 채널 IRP 계산
- 각 후보 채널별로 actual_onset ~ +10초 구간의 최저 400샘플(4초) 평균 계산
- 환자별 중앙값(median) 사용
- 안정압 < 5mmHg 채널 제외 (위 신호)

#### 단계 5: 최종 선택
- IRP 최솟값 채널 = LES
- 근거: LES는 삼킴 시 가장 깊이 이완되는 괄약근

**핵심 개선점**:
- ❌ 제거: 횡격막 각 진동 필터 (복잡하고 불안정)
- ✅ 추가: 첫 도달 단조 진행 원칙 (생리학적으로 타당)
- ✅ 추가: IRP 기반 최종 선택 (기능적 정의 반영)

**결과 요약**:
```
선택된 LES 채널 분포:

- ch29: 0명
- ch30: 7명
- ch31: 11명
- ch32: 8명
- ch33: 4명
```

**주요 발견**:
1. **정상 범위**: 대부분 ch30~33에서 선택됨 (해부학적으로 타당)
2. **ch34~35 제외**: 알고리즘이 위(gastric) 채널을 LES로 선택하지 않음 (의도대로 작동)
3. **특이 사례**:
   - **A17**: 이중 삼킴이 많아 IRP 유효값 적음 (알고리즘 문제 아님, 데이터 특성)
   - **A23, A29, A31**: LES 고압대 자체가 약함 (안정압 < 10mmHg)
   - 이들도 알고리즘은 정상 작동, 단지 생리적으로 LES 압력이 낮은 환자

**외부 압박 채널 필터링 성공**:
- 예시: A14 ch22 (대동맥 압박)
- 삼킴 전 구간에도 높고 삼킴 후에도 높음
- `최대압 > 기저선+15mmHg` 조건으로 자동 제외됨

### LES 선택 알고리즘 핵심 코드

```python
# 단계 1: 참조 삼킴 선택 (상위 3개)
distal_scores = [_distal_end(np.asarray(c)) for _, c in all_crops]
_ao_list = [find_actual_onset(np.asarray(c)) for _, c in all_crops]
_valid_idx = [i for i, (ao, dbl) in enumerate(_ao_list) if ao is not None and not dbl]
top_idx = sorted(_valid_idx, key=lambda i: distal_scores[i], reverse=True)[:3]
crops_ref = [np.asarray(all_crops[i][1]) for i in top_idx]

# 단계 2: 첫 도달 단조 진행
MIN_DELAY = int(1.0 * SR)  # 수축파는 UES 이후 최소 1초 뒤
TOLERANCE = 0.5 * SR
distal_end_ch = SEARCH_START
prev_t = fh_mean.get(SEARCH_START)
miss = 0
for ch in range(SEARCH_START + 1, 35):
    t = fh_mean.get(ch)
    if t is None:
        miss += 1
        if miss >= 2: break
        continue
    if prev_t is None or t >= prev_t - TOLERANCE:
        distal_end_ch = ch; prev_t = t; miss = 0
    else:
        miss += 1
        if miss >= 2: break

# 단계 3: LES 탐색 범위
les_lo = max(distal_end_ch + 1, 28)
les_hi = min(les_lo + 4, 34)

# 단계 4~5: IRP 기반 선택
ch_irp = {}
for ch in les_cands:
    irps = []
    for c in crops:
        ao, dbl = find_actual_onset(c)
        if ao is None or dbl: continue
        irp_end = ao + int(10 * SR)
        if irp_end > len(c): continue
        irps.append(float(np.sort(c[ao:irp_end, ch])[:400].mean()))
    ch_irp[ch] = float(np.median(irps)) if irps else float('inf')

valid_irp = {ch: v for ch, v in ch_irp.items()
             if v < float('inf') and ch_pre[ch] >= 5.0}
best = min(valid_irp, key=valid_irp.get)
```
